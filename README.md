## 📖 项目简介

短链接（Short Link）是指将一个原始的长URL（Uniform Resource Locator）通过特定的算法或服务转化为一个更短、易于记忆的URL。短链接通常只包含几个字符，而原始的长URL可能会非常长。

### 短链接的原理

1. **生成唯一标识符**：当用户输入或提交一个长URL时，短链接服务会生成一个唯一的标识符或者短码
2. **将标识符与长URL关联**：短链接服务将这个唯一标识符与用户提供的长URL关联起来，并将其保存在数据库或者其他持久化存储中
3. **创建短链接**：将生成的唯一标识符加上短链接服务的域名作为前缀，构成一个短链接
4. **重定向**：当用户访问该短链接时，短链接服务接收到请求后会根据唯一标识符查找关联的长URL，然后将用户重定向到这个长URL
5. **跟踪统计**：一些短链接服务还会提供访问统计和分析功能，记录访问量、来源、地理位置等信息

### 短链接的应用场景

短链接经常出现在日常生活中，例如：
- **营销推广**：企业在营销活动中通过短链接识别用户行为、点击率等关键信息
- **社交媒体**：在微博、微信等字符受限的场景中分享长链接
- **短信推广**：节省短信字符，提高用户体验
- **数据分析**：通过短链接收集用户访问数据，进行精准营销

### 主要作用

- **提升用户体验**：用户更容易记忆和分享短链接，增强了用户的体验
- **节省空间**：短链接相对于长URL更短，可以节省字符空间，特别是在一些限制字符数的场合，如微博、短信等
- **美化链接**：短链接通常更美观、简洁，不会包含一大串字符
- **统计和分析**：可以追踪短链接的访问情况，了解用户的行为和喜好

## 🔥 技术亮点

### 1. 海量并发处理
- **Redis缓存**：缓存热点数据，提升访问速度
- **布隆过滤器**：防止缓存穿透，降低数据库压力
- **分布式锁**：保证并发安全性
- **读写分离**：主库负责写，从库负责读

### 2. 海量数据存储
- **分库分表**：按用户ID分库，按时间分表，支持亿级数据存储
- **水平扩展**：支持动态增加数据库节点

### 3. 多租户支持
- **数据隔离**：不同用户的数据完全隔离
- **独立管理**：每个用户独立管理自己的短链接

### 4. 高可用设计
- **微服务架构**：服务拆分，独立部署和扩展
- **消息队列**：异步处理，解耦系统
- **限流熔断**：防止系统过载

### 5. 完善的监控
- **访问日志**：记录每次访问的详细信息
- **统计数据**：多维度统计分析
- **数据可视化**：直观展示统计结果

## 🚀 快速开始

### 环境要求

- JDK 17+
- Maven 3.6+
- MySQL 8.0+
- Redis 6.0+
- RabbitMQ 3.12+
- Node.js 16+ (前端)

### 安装步骤

1. **克隆项目**
```bash
git clone [项目地址]
cd short
```

2. **启动后端服务**
```bash
# 启动网关服务
cd gateway
mvn spring-boot:run

# 启动管理后台服务
cd admin
mvn spring-boot:run

# 启动聚合服务
cd project
mvn spring-boot:run
```

3. **启动前端服务**
```bash
cd console-vue
npm install
npm run dev
```

4. **访问系统**
- 前端地址：http://localhost:5173
- 网关地址：http://localhost:8080
